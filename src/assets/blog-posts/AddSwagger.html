The basic architecture for the current web is as follows:

  1. A client (user) downloads a front side app to their browser. Typically this client code is limited to displaying what the user sees on screen and performing any tasks that do need access to company servers.
  2. Sensitive user and company data is typically stored on the companies servers. This of course is for control and security reasons.
  3. Naturally the client will need to access and add to this data at some point. To do access this data the client must communicate to the backend through HTTP requests

Companies therefore must write backend applications which must know how to handle the requests the client makes. The front end must interface with the company backend through an Application Specific Interface (API). Obviously the developers building the backend should document and test this API so the front end developers know what methods to call and how to use them. This is where SwaggerUI is useful. It provides a simple interface to view the API, see what objects each method takes and returns. It also offers the ability to test the methods themselves by allowing you to execute the methods without having to write any front end code.

Add Swagger to ASP.NET Core Project

  1. Create new project

    dotnet new webapp -o MySwaggerProject

  2. Add required packages

    dotnet add package Swashbuckle.AspNetCore

  3. Include library in Startup.cs

    using Microsoft.OpenApi.Models;

  4. Add to services in Startup.cs

    services.AddSwaggerGen();

  5. Configure app

    app.UseSwagger();
app.UseSwaggerUI(c => {
c.SwaggerEndpoint("/swagger/v1/swagger.json", "My API V2");
});

  6. Done! Just run the app

    dotnet run

Finally you can access Swagger UI from the following URL

https://<app url>/swagger
